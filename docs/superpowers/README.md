# superpowers 文档归档规范

所有通过 superpowers 技能(`brainstorming`、`writing-plans` 等)产生的需求/设计文档、实施方案,统一保存在本目录下,按类别归档:

- `specs/` —— `brainstorming` 产出的需求/设计文档,文件名格式 `YYYY-MM-DD-<topic>-design.md`
- `plans/` —— `writing-plans` 产出的实施方案,文件名格式 `YYYY-MM-DD-<feature-name>.md`

这是硬性要求。即便处于 Plan Mode、因文件写入限制只能先把设计写进 plan 文件(`~/.claude/plans/` 或项目内 `.claude/plans/`),approve 之后也必须把最终设计/计划补一份到上面对应目录下,不能只留在 plan 文件里。
