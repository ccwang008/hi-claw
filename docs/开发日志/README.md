# 开发日志

记录 my-claw 🦞 的开发过程,分两类归档:

- `vibecoding日志/` —— 每一次开发的对话记录。一次开发(一段完整的编码会话)一个文件,文件名格式 `YYYY-MM-DD-<主题>.md`。
- `提交日志/` —— 每一次 GitHub 提交的记录。一次提交一个文件,文件名格式 `YYYY-MM-DD-<commit短hash>-<主题>.md`。

## 约定

- 日期用当天实际日期,同一天多次开发/提交用主题区分。
- vibecoding 日志侧重「做了什么、为什么这么做、遇到的问题」;提交日志侧重「本次提交的改动清单与 commit 信息」。
- 两者可互相引用:提交日志里注明对应的 vibecoding 日志,反之亦然。
