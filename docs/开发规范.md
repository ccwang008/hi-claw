# 开发规范

## 测试策略(现阶段:2026-07-04 起)

**现阶段开发新功能时,暂时跳过测试:不要求为新功能编写测试代码,先把功能实现出来,测试留到后续统一补齐。**

- 不需要按 TDD 先写失败测试再实现
- 不需要为新功能补充测试用例
- 例外:如果改动导致**已有测试**断言过时(例如新增工具后,校验工具名单的断言需要同步更新),仍需修正,以保证已有测试套件能跑通——这不算"新写测试",只是维持现状不被破坏

**这是阶段性状态,以后会重新开启"测试先行"的要求。** 到时更新本文档,并在 superpowers 相关的 spec/plan 文档里恢复完整的测试计划。
